
**در چه شرایطی از signal استفاده می‌کنیم؟**

در شرایطی که می‌خواهیم یک مقدار reactive داشته باشیم که تغییرات آن به صورت خودکار در UI یا سایر بخش‌های برنامه منعکس شود، استفاده از signal بسیار مفید است.
چون Angular تغییرات آن را track می‌کند و مقادیر وابسته مثل `computed()` به صورت خودکار reactive آپدیت می‌شوند، ولی متغیر معمولی نیاز به مدیریت دستی دارد.

**در چه شرایطی از تابع effect() استفاده می‌کنیم؟**

وقتی می‌خواهیم یک side effect را در پاسخ به تغییرات یک signal یا computed ایجاد کنیم، از تابع `effect()` استفاده می‌کنیم. این تابع به ما اجازه می‌دهد تا کدی را اجرا کنیم که به صورت خودکار هر بار که سیگنال‌های وابسته تغییر می‌کنند، اجرا شود.

این تابع به صورت خودکار وابستگی‌ها را track می‌کند و هر بار که سیگنال‌های مورد استفاده در آن تغییر کنند، کد داخل `effect()` اجرا خواهد شد. این برای مواردی مانند به‌روزرسانی DOM، فراخوانی API یا هر عملیاتی که نیاز به واکنش به تغییرات داده‌ها دارد، بسیار مفید است.
